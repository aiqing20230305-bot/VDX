## 测试场景

### 场景1：创建角色

**流程**：
1. 用户上传角色参考图片（支持格式：JPG/PNG/WEBP）
2. 填写角色信息（名称、描述、标签）
3. 点击创建按钮
4. 系统自动提取角色特征（Claude Vision + OpenAI Embeddings）
5. 角色保存到数据库

**验证点**：
- [ ] 图片上传成功
- [ ] 表单验证正确（名称必填、图片必传）
- [ ] 特征提取成功（face/body/style）
- [ ] 角色出现在角色库列表
- [ ] 返回正确的角色ID

**预期结果**：
- 角色创建时间 < 30秒（包含特征提取）
- 返回完整的角色对象，包含features字段
- 角色ID格式正确（cuid）

---

### 场景2：查询和搜索角色

**流程**：
1. 打开角色库界面
2. 测试搜索功能（输入关键词）
3. 测试标签筛选（选择标签）
4. 查看角色详情

**验证点**：
- [ ] 角色列表正确显示
- [ ] 搜索结果准确（语义搜索）
- [ ] 标签筛选工作正常
- [ ] 详情页展示完整特征
- [ ] 统计信息正确（使用次数）

**预期结果**：
- 查询响应时间 < 500ms
- 语义搜索返回相似度分数
- 标签筛选结果正确
- 详情页布局正常

---

### 场景3：使用角色生成分镜

**流程**：
1. 创建或选择测试脚本
2. 选择生成模式（全自动/步骤审核）
3. 系统提示：是否使用角色一致性？
4. 点击"选择角色"打开角色库
5. 选择角色并确认
6. 系统生成分镜（应用角色约束）

**验证点**：
- [ ] 角色选择模态框正常打开
- [ ] 角色选择后状态更新
- [ ] characterId正确传递给API
- [ ] 生成的分镜提示词包含角色特征
- [ ] 所有帧应用相同的角色约束

**预期结果**：
- 角色选择流程顺畅
- 分镜生成时显示"使用角色一致性"提示
- 提示词中注入角色关键特征（发型、体型、服装、配色）

---

### 场景4：跳过角色选择

**流程**：
1. 创建脚本后选择模式
2. 点击"跳过"按钮
3. 系统直接生成分镜（无角色约束）

**验证点**：
- [ ] 跳过按钮正常工作
- [ ] 生成分镜时不传递characterId
- [ ] 提示词中不包含角色特征

**预期结果**：
- 流程不被阻塞
- 正常生成分镜
