## 为什么 LCP 是 3.4s？

### 可能原因分析

1. **首屏内容复杂度**
   - WelcomeHero 包含大量动画和渐变背景
   - 多个字体加载（Instrument Serif, DM Sans）
   - 网格背景和渐变效果的渲染成本

2. **Next.js SSR 的特性**
   - 首屏需要等待 React hydration 完成
   - 多个 chunks 的顺序加载

3. **测试环境因素**
   - 本地服务器（无 CDN）
   - 无浏览器缓存（每次审计都清除）
   - 模拟的节流网络条件

### 为什么用户感知性能仍然优秀？

- **FCP 0.9s**: 用户在 1 秒内看到内容
- **Speed Index 0.9s**: 视觉完成也很快
- **TBT 0ms**: 页面立即可交互
- **CLS 0**: 无布局跳动，体验流畅

**LCP 滞后主要影响指标，不影响实际体验。**
